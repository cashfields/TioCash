PSL - Prime Suspect Level - count consecutively within a *fam


Inside each *fam where PSL = 0 is a unique set of sub-keys
The sub-keys can not be applied to any other *fam
The PSL = 0 is the first level , and the unique keys are generated here
Only these sub-keys will satisfy this PS at this level
We have sorted off all the other possible solutions
Note that the *fam sub-key of 1 is equal to the *fam in the table
Note how the sub-keys are balanced thru the table symmetrically to make 30

Find the square root of PS

Use the sub-keys and loop to the right: sub-key + 30

Add 'just another 30 away' until
a) a factor is found
or
b) the square root of PS is reached

If a factor is found = non-prime

If no factor is found = prime


Think in these terms , we are taking a slice of numbers from 0 to 29 , test the PS to see which *wildcard , then which *fam. Work thru the found *fam and loop thru until the sq root ; using the sub-keys. If test with mod 1 for 0 , then composite number. If by the time we hit the sq root – then prime.

Since we have slowly eliminated number by sorting , and more sorting all that can be left is prime. This is a backdoor approach , find and eliminate all non-prime numbers and the last sort is *fam1. If there are no other solutions to the PS in the loop , then the default is *fam1 = prime.
